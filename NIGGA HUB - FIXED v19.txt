What was killing it: line 197 had
UserInputService:GetPropertyChangedSignal("LastInputType")
​
LastInputType isn't a real Roblox property, and GetPropertyChangedSignal validates the name, so it threw "LastInputType is not a valid property name" and crashed the entire script before any UI built. That's why you got nothing on Delta but the diagnostic. (Potassium/PC happened to be lenient about it; Delta is strict — which is actually the correct behavior.)
The fix: I swapped it for the real event:
UserInputService.LastInputTypeChanged:Connect(function(lastType)
    if (lastType or UserInputService:GetLastInputType()) ~= Enum.UserInputType.Touch then reassertControls() end
end)
​
Same behavior (re-assert touch controls when input switches away from touch), but using a valid API that works on every executor. Your GETHUI panel showing screen GUIs was just the executor's own UI container — normal.
